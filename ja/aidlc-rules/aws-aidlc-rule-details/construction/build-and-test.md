# ビルドとテスト

**目的**: すべての作業単位(unit of work)をビルドし、包括的なテスト戦略を実行する

## 前提条件
- すべての作業単位においてコード生成が完了していること
- すべてのコード成果物が生成済みであること
- プロジェクトがビルドおよびテストの実施可能な状態であること

---

## ステップ 1: テスト要件の分析

プロジェクトを分析し、適切なテスト戦略を決定する。
- **ユニットテスト**: コード生成時に各作業単位ごとに生成済み
- **インテグレーションテスト**: 作業単位またはサービス間の相互作用を検証
- **パフォーマンステスト**: 負荷・ストレス・スケーラビリティのテスト
- **エンドツーエンドテスト**: ユーザーワークフロー全体の検証
- **コントラクトテスト**: サービス間の API コントラクト検証
- **セキュリティテスト**: 脆弱性スキャン、ペネトレーションテスト

## ステップ 9: ユーザーへの結果報告

以下の構成で完了メッセージを提示する。
     1. **完了アナウンス**（必須）: 必ず以下から開始すること:

```markdown
# 🔨 Build and Test Complete
```

     2. **AI サマリー**（任意）: ビルドおよびテスト結果の箇条書きサマリーを提示する
        - 形式: "Build and test has completed with the following results:"
        - ビルドステータスと成果物を列挙する
        - テスト結果をカテゴリ別（ユニット、インテグレーション、パフォーマンス等）に列挙する
        - 生成された手順書ファイルを列挙する
        - ワークフロー指示（"please review"、"let me know"、"proceed to next phase"、"before we proceed" 等）は含めないこと
        - 事実に基づき、内容に集中すること
     3. **フォーマット済みワークフローメッセージ**（必須）: 必ず以下の形式で終了すること:

```markdown
> **📋 <u>**REVIEW REQUIRED:**</u>**  
> Please examine the build and test summary at: `aidlc-docs/construction/build-and-test/build-and-test-summary.md`



> **🚀 <u>**WHAT'S NEXT?**</u>**
>
> **You may:**
>
> 🔧 **Request Changes** - Ask for modifications to the build and test instructions based on your review
> ✅ **Approve & Continue** - Approve build and test results and proceed to **Operations**

---
```
