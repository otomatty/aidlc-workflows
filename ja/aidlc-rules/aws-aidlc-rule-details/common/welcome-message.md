# 👋 AI-DLC（AI駆動開発ライフサイクル）へようこそ！ 👋

あなたの特定のニーズに合わせてインテリジェントに適応するアダプティブなソフトウェア開発ワークフロー(workflow)を通じてご案内します。

## AI-DLC とは？

AI-DLC(AI駆動開発ライフサイクル)は、プロジェクトのニーズに適応する構造化された柔軟なソフトウェア開発プロセスです。経験豊富なソフトウェアアーキテクトが次のことをしてくれるとイメージしてください：

- **要件を分析し**、必要に応じて確認の質問をする
- **複雑さとリスクに基づいて最適なアプローチを計画する**
- **シンプルな変更には不必要なステップをスキップし**、複雑なプロジェクトには包括的なカバレッジを提供する
- **すべてを文書化し**、決定と根拠の完全な記録を残す
- **明確なチェックポイントと承認ゲートで各フェーズを通じてガイドする**

### フェーズの概要：

**INCEPTION PHASE** - *計画とアプリケーション設計*
- **目的**：何を作るか（WHAT）、なぜ作るか（WHY）を決定する
- **活動**：要件の理解、既存コードの分析（ある場合）、アプローチの計画
- **成果物**：明確な要件(requirements)、実行計画、並行開発のための作業単位(unit of work)数の決定
- **あなたの役割**：質問への回答、計画のレビュー、方向性の承認

**CONSTRUCTION PHASE** - *詳細な設計、実装とテスト*
- **目的**：どのように作るか（HOW）を決定する
- **活動**：詳細な設計（必要な場合）、コード生成、包括的なテスト
- **成果物**：動作するコード、テスト、ビルド手順
- **あなたの役割**：設計のレビュー、実装計画の承認、結果の検証

**OPERATIONS PHASE** - *デプロイメントとモニタリング（将来）*
- **目的**：どのようにデプロイ・運用するか（DEPLOY and RUN）
- **ステータス**：将来のデプロイメントおよびモニタリングワークフロー(workflow)のプレースホルダー
- **現在の状態**：ビルドとテストの活動はCONSTRUCTION PHASEで処理される

## 主要な原則：

- ⚡ **完全にアダプティブ**：各ステージはあなたのニーズに基づいて独立して評価される
- 🎯 **効率的**：シンプルな変更は必須ステージのみを実行する
- 📋 **包括的**：複雑な変更はすべての安全策を伴う完全な処理を受ける
- 🔍 **透明性**：作業開始前に実行計画を確認・承認できる
- 📝 **文書化**：すべての決定と変更の完全な監査証跡
- 🎛️ **ユーザーコントロール**：ステージの追加や除外をリクエストできる

## 次に起こること：

1. **ワークスペースを分析します**：新しいプロジェクトか既存のプロジェクトかを把握するため
2. **要件(requirements)を収集します**：必要に応じて確認の質問をします
3. **実行計画を作成します**：実行を提案するステージとその理由を示します
4. **計画を確認・承認していただきます**（または変更をリクエストできます）
5. **計画を実行します**：各主要ステージでチェックポイントを設けます
6. **動作するコードをお届けします**：完全なドキュメントとテストとともに

AI-DLC(AI駆動開発ライフサイクル)プロセスは以下に適応します：
- 📋 あなたの意図の明確さと複雑さ
- 🔍 既存のコードベースの状態
- 🎯 変更のスコープと影響
- ⚡ リスクと品質の要件

それでは始めましょう！
