# ビジョンドキュメント：CalcEngine 関数電卓 API

## エグゼクティブサマリー

CalcEngine は、REST API として提供されるホスト型の関数電卓ライブラリです。ソフトウェア開発チームが計算ロジックを自社で構築・保守することなく、正確で標準に準拠した数学的計算を自分たちのアプリケーションに組み込めるようにします。エンジニアリングチームが自社製品の中核とは無関係な数学関数の実装・テスト・デバッグに数か月を費やしてしまうという繰り返し発生する問題を解決します。教育技術、金融モデリング、エンジニアリングシミュレーション、データ分析分野で開発を行う開発者を獲得することで、24か月以内にARR $200万を達成するサブスクリプション型APIサービスの実現を目指します。

## ビジネス背景

### 課題の定義

教育、金融、エンジニアリング、科学分野の製品を構築するソフトウェアチームは、基本的な算術演算を超えた数学的計算を定期的に必要とします。彼らの選択肢は二つです。ローカルライブラリを使用する（言語依存が多く、プラットフォーム間で一貫性がなく、保守の負担も大きい）か、自前で数学関数を構築する（コストが高く、エラーが発生しやすく、時間がかかる）かです。

具体的な問題点は以下の通りです。

- **精度リスク**：数学の専門知識を持たないチームは、微妙な浮動小数点誤差、不正確なエッジケース処理（ゼロ除算、オーバーフロー、定義域エラー）、および一貫性のない丸め動作を引き起こし、製品への信頼を損ないます。
- **作業の重複**：三角関数、統計分布、行列演算、単位変換を必要とするチームはそれぞれ独自に実装します。この作業が数千もの企業で繰り返されています。
- **クロスプラットフォームの不整合**：Pythonバックエンドで行われた計算は、JavaScriptフロントエンドでの同じ計算と異なる結果をもたらす場合があります。複数プラットフォームにまたがって運営する顧客は一貫性を保証できません。
- **コンプライアンスと監査可能性**：規制産業（金融、医療、エンジニアリング）では、計算はトレーサブルで、バージョン管理され、検証されていなければなりません。場当たり的な実装は監査要件を満たすことがほとんどありません。

### ビジネスの推進要因

- **APIファーストの経済圏**：開発者はライブラリを組み込むよりもホスト型APIを利用することをますます好んでいます。Stripe（決済）、Twilio（通信）、SendGrid（メール）がこのモデルを実証しました。数学的計算に相当するものは存在しません。
- **EdTech の成長**：グローバルな教育技術市場は急速に拡大しており、計算機能は数学、理科、工学のコースウェア全体で普遍的な要件です。
- **規制上の圧力**：金融サービス企業は計算精度に対する監視が強まっており、認定された監査可能なAPIはコンプライアンスの負担を軽減します。
- **AI/ML の前処理**：データサイエンスチームはモデルトレーニング前の前処理ステップとして、信頼性の高い数学的変換を必要とします。精度と再現性を保証するAPIはMLパイプラインで明確な価値を持ちます。

### ビジネス上の制約

- **ブートストラップ予算**：初期開発は既存の収益から資金調達。MVPに向けた外部投資は予定なし。MVPの総予算は $150K（開発、インフラ、初期マーケティングを含む）。
- **小規模チーム**：バックエンド開発者2名、フロントエンド開発者1名（ドキュメントポータル用）、QAエンジニア1名。MVPフェーズでは専任の数学博士を採用しない。
- **市場投入までの時間**：フェーズ2への投資を確約する前に需要を検証するため、MVPは6か月以内に公開する必要があります。
- **価格モデル**：無料ティア（普及促進のため）と使用量ベースの有料ティアをサポートする必要があります。ローンチ時に料金体系のインフラが必要です。
- **精度基準**：すべての関数はIEEE 754倍精度演算の精度を満たすか、それを上回る必要があります。任意精度モードはフェーズ2の機能であり、MVPには含まれません。

## 全体スコープのビジョン

### 製品ビジョンの説明

CalcEngineは、StripeがPaymentのデファクトスタンダードになったように、基本的な算術演算を超えた数学的演算を必要とするあらゆるアプリケーションにおける、デフォルトの計算レイヤーとなります。自分で構築するよりも正確で、一貫性があり、統合しやすいAPIを提供することで、それを実現します。

### 機能エリア

#### 機能エリア1：基本算術と代数

- **説明**：標準的な言語数学ライブラリが確実に提供する以上の、基本的な数学演算。
- **主な機能**：
  - 任意精度の整数および小数演算（最大1,000桁の精度設定が可能）
  - 式の解析と評価（"2 * sin(pi/4) + log(100)" などの文字列式を受け付ける）
  - 多項式演算（評価、根の発見、因数分解）
  - 方程式の解法（線形、二次、三次、連立線形方程式）
  - 分数および有理数演算（厳密な表現、簡約化）
  - 複素数演算（加算、乗算、極形式/直交形式変換）
- **ユーザー価値**：開発者は数式を文字列として送信し、パーサーの実装や演算子の優先順位、浮動小数点の誤差、エッジケースを気にすることなく、正確で検証済みの結果を受け取れます。

#### 機能エリア2：三角法と幾何学

- **説明**：完全な三角法と幾何学的計算機能。
- **主な機能**：
  - 6つすべての三角関数とその逆関数（sin、cos、tan、csc、sec、cot）
  - 双曲線関数とその逆関数
  - 度数・ラジアン・グラジアン変換
  - 座標系変換（直交、極、球面、円柱）
  - 幾何学的計算（標準的な図形の面積、体積、周長）
  - 2次元および3次元空間での距離と角度の計算
- **ユーザー価値**：三角関数のエッジケース（特定の角度での厳密な値、象限の処理、定義域の検証）を実装する必要をなくします。

#### 機能エリア3：統計と確率

- **説明**：統計分析と確率分布関数。
- **主な機能**：
  - 記述統計（平均、中央値、最頻値、分散、標準偏差、四分位数、百分位数）
  - 確率分布（正規分布、二項分布、ポアソン分布、カイ二乗分布、t分布、F分布）のPDF、CDF、逆CDF
  - 回帰分析（線形、多項式、指数、対数）
  - 仮説検定（t検定、カイ二乗検定、分散分析）
  - 組み合わせ論（順列、組み合わせ、階乗、二項係数）
  - 設定可能な分布とシードを使った乱数生成
- **ユーザー価値**：1回のAPIコールで統計ライブラリのインポートと設定を代替できます。結果は再現可能で監査可能です。

#### 機能エリア4：線形代数と行列演算

- **説明**：エンジニアリング、グラフィックス、データサイエンス向けの行列・ベクトル計算。
- **主な機能**：
  - 行列演算（加算、乗算、スカラー演算）
  - 行列分解（LU、QR、SVD、コレスキー、固有値）
  - 行列式、逆行列、ランク、トレース
  - ベクトル演算（内積、外積、正規化）
  - 連立線形方程式（ガウスの消去法、最小二乗法）
  - 大規模問題向けの疎行列サポート
- **ユーザー価値**：シミュレーション、ML、グラフィックスアプリケーションを構築するチームは、LAPACKへのリンクやネイティブバインディングの保守なしに、検証済みの線形代数を利用できます。

#### 機能エリア5：微積分

- **説明**：記号的・数値的な微積分演算。
- **主な機能**：
  - 数値微分（1次および高次の導関数）
  - 数値積分（設定可能な手法による定積分：台形法、シンプソン法、ガウス求積法）
  - 記号微分・積分（対応する式の種類に対して）
  - 極限と級数展開（テイラー展開、マクローリン展開）
  - 常微分方程式ソルバー（オイラー法、ルンゲ・クッタ法）
- **ユーザー価値**：エンジニアや科学者は、コンピュータ代数システムを組み込むことなく、API経由で微積分演算を利用できます。

#### 機能エリア6：単位変換と物理定数

- **説明**：標準的な単位変換と検証済みの物理・数学定数へのアクセス。
- **主な機能**：
  - すべてのSI単位および一般的なヤード・ポンド法単位間の変換（長さ、質量、温度、時間、エネルギー、圧力、速度など）
  - 通貨換算（金融データプロバイダーからの日次レート更新）
  - 物理定数（光速、プランク定数、アボガドロ数など）、出典と不確実性の値付き
  - 設定可能な精度での数学定数（pi、e、黄金比など）
  - 次元解析（単位の組み合わせが物理的に意味を持つかの検証）
- **ユーザー価値**：1つのAPIが複数の変換ライブラリとハードコードされた定数値を代替し、定数の出典が確かで最新であることを保証します。

#### 機能エリア7：金融数学

- **説明**：貸付、投資、リスク分析向けの金融計算関数。
- **主な機能**：
  - 貨幣の時間的価値（現在価値、将来価値、年金、永久年金）
  - ローン償却スケジュール
  - 債券価格と利回り計算
  - オプション価格付け（ブラック・ショールズ、二項モデル）
  - 内部収益率（IRR）と正味現在価値（NPV）
  - 減価償却方法（定額法、定率法、級数法）
- **ユーザー価値**：FinTech企業は、独自の数学エンジンを構築することなく、監査可能で規制対応の金融計算を利用できます。

#### 機能エリア8：開発者体験とプラットフォーム

- **説明**：CalcEngineを採用しやすくするAPIプラットフォーム、ドキュメント、SDK、開発者ツール。
- **主な機能**：
  - ライブ「試用」サンドボックスを備えたインタラクティブなAPIドキュメント
  - Python、JavaScript/TypeScript、Java、C#、Go、Ruby向けのクライアントSDK
  - 長時間実行する計算向けのWebhookサポート（バッチ処理）
  - APIキーごとの計算履歴と監査ログ
  - 明確なクォータとオーバーエージ処理を備えたレート制限
  - 12か月の廃止予告ポリシーを持つAPIバージョニング
  - チーム向けワークスペース機能（共有APIキー、使用状況ダッシュボード、請求管理）
- **ユーザー価値**：開発者はサインアップから5分以内に最初のAPIコールを成功させることができます。

### 統合ポイント

- **決済プロセッサー**（Stripe） - サブスクリプション課金と使用量ベースの計量
- **IDプロバイダー**（Auth0またはCognito） - 開発者アカウントの認証
- **金融データプロバイダー**（通貨レート用） - 日次の為替レートフィード
- **NIST / CODATA** - 物理定数の信頼できる情報源
- **CI/CDシステム**（GitHub Actions、GitLab CI） - SDKの公開とバージョン管理
- **モニタリング**（DatadogまたはCloudWatch） - APIパフォーマンス、エラーレート、使用状況ダッシュボード

### ユーザージャーニー（全体ビジョン）

#### ジャーニー1：EdTech開発者がコースプラットフォームに計算機能を追加する

1. 開発者が「scientific calculator API」を検索してCalcEngineを発見し、ドキュメントサイトにたどり着く。
2. 開発者が無料アカウントを作成し、2分以内にAPIキーを取得する。
3. 開発者がインタラクティブなドキュメントを参照し、三角関数エンドポイントを見つけ、サンドボックスで `sin(pi/4)` をテストする。
4. 開発者がpip経由でPython SDKをインストールし、学生が入力した式をCalcEngineに送信して結果を表示する3行の統合コードを書く。
5. 開発者がAPIをステップバイステップの解法解説を返すように設定し、学生が答えの導き方を確認できるようにする。
6. コースプラットフォームが公開される。毎日何千人もの学生が計算を送信する。開発者はCalcEngineのダッシュボードで使用状況を監視し、無料ティアの上限に達したら有料ティアにアップグレードする。

**成果**：教育プラットフォームは、数学パーサーの構築とテストに数週間費やす代わりに、午後一日で信頼性の高い計算機機能をリリースします。

#### ジャーニー2：FinTechスタートアップがローン比較ツールを構築する

1. 貸付スタートアップのプロダクトチームが、顧客向けローン比較ツールのために償却スケジュール、APR計算、現在価値計算を必要としている。
2. 開発者がCalcEngineにサインアップし、金融数学セクションに移動する。
3. 開発者がローン償却エンドポイントを使用して、金利6.5%の30年ローンの返済スケジュールを生成する。APIは月ごとの元金、利息、残高の内訳を返す。
4. 開発者がNPVとIRRエンドポイントを統合して、顧客が複数のローンオファーを並べて比較できるようにする。
5. コンプライアンスチームがCalcEngineの精度認定と監査ログを確認する。各計算はタイムスタンプ付きの入出力を持つバージョン管理されたAPIコールに追跡可能である。
6. ローン比較ツールがローンチされる。CalcEngineは月50万件の計算を処理する。スタートアップは使用量に応じて支払い、クオンツ開発者の採用を避けられる。

**成果**：FinTechスタートアップは独自の計算ロジックを構築することなく、コンプライアンス対応の監査可能な金融ツールをローンチします。

#### ジャーニー3：データサイエンティストがMLパイプラインでCalcEngineを使用する

1. 医療企業のデータサイエンティストが、モデルトレーニング前の前処理として患者測定データを統計的変換（Zスコア、百分位ランク、対数変換）を使って正規化する必要がある。
2. データサイエンティストがCalcEngineのPython SDKをインストールし、Jupyterノートブック内から統計エンドポイントを呼び出す。
3. SDKは値の配列を受け取り、記述統計と変換済みデータセットを返す。
4. データサイエンティストがバッチモードを設定して10万件のレコードを処理する。CalcEngineは処理完了時にWebhook経由で結果を返す。
5. CalcEngineが再現可能な結果（同じ入力、同じ出力、バージョンをまたいで）を保証するため、科学者は再現性のために研究論文でAPIバージョンを引用できる。

**成果**：科学者はカスタム統計コードを書いてデバッグすることなく、検証済みで再現可能な統計変換を利用できます。

### スケーラビリティと成長

- **地理的展開**：US-Eastに初期展開。ユーザーの地理データに基づき、MVPから12か月以内にEU-WestとAP-Southeastに展開。
- **ボリューム成長**：3年以内に月10億APIコールに対応できるアーキテクチャを構築。サーバーレスから開始し、レイテンシが問題になる場合は高トラフィックエンドポイントをコンテナに移行。
- **機能の成長**：顧客需要データに基づいて新しい機能エリアを追加。候補には数論、グラフ理論、信号処理、最適化ソルバーが含まれる。
- **エンタープライズ展開**：データを共有APIに送信できない規制産業向けに、オンプレミスデプロイオプションを導入。フェーズ3を対象とする。
- **マーケットプレイスへの出品**：追加の配布チャネルとしてAWS Marketplace、Azure Marketplace、RapidAPIへの出品。

## MVPスコープ

### MVPの目的

ローンチ時に中核となる数学関数を提供し、無料ティアのサインアップを通じて採用を検証し、ローンチから6か月以内に少なくとも50アカウントを有料プランに転換することで、開発者がホスト型の関数電卓APIに対して料金を支払う意思があることを証明します。

### MVPの成功基準

- [ ] ローンチから3か月以内に登録開発者アカウント1,000件
- [ ] 3か月以内に月間アクティブAPI利用者（月1回以上のコール）300名
- [ ] 6か月以内に有料サブスクライバー50件
- [ ] 6か月以内にMRR $15K
- [ ] 最初の3か月間でAPI稼働率99.9%
- [ ] MVPのすべてのエンドポイントで平均レスポンスタイム（p50）50ms未満
- [ ] 重大な精度バグの報告ゼロ件（誤った計算結果を返すもの）
- [ ] 3か月時点での開発者アンケートにおけるNPS 40以上

### MVPユーザージャーニー

#### ジャーニー1：開発者がCalcEngineを発見して統合する

1. 開発者が「math expression evaluation API」を検索し、CalcEngineのドキュメントを見つける。
2. 開発者が「APIキーを取得」をクリックし、1ページのサインアップフォーム（メールアドレス、パスワード、会社名は任意）に記入する。
3. 開発者が確認ページと歓迎メールで即座にAPIキーを受け取る。
4. 開発者がドキュメントからcURLの例をコピーし、ターミナルで実行する：`curl -X POST https://api.calcengine.io/v1/evaluate -H "Authorization: Bearer {key}" -d '{"expression": "sin(pi/4) * 2 + sqrt(16)"}'`
5. 開発者がJSONレスポンスを受け取る：`{"result": 5.414213562373095, "expression": "sin(pi/4) * 2 + sqrt(16)", "precision": "double"}`
6. 開発者がドキュメントサイトのPythonコード例を読み、アプリケーションにコピーし、式の文字列をユーザー入力に置き換える。
7. アプリケーションが公開される。開発者はCalcEngineのダッシュボードで使用状況を監視する。

**成果**：開発者は発見から動作する統合まで15分以内に完了します。
**全体ビジョンとの差異**：SDKなし（生のHTTPコール）、ステップバイステップの解法なし、監査ログなし。

#### ジャーニー2：EdTech企業が学生向けにCalcEngineを評価する

1. EdTechのプロダクトマネージャーが、宿題チェック機能のためにCalcEngineを評価するよう開発者に依頼する。
2. 開発者が無料ティアにサインアップし、APIサンドボックスで20件の一般的な学生の計算（二次方程式の公式、三角関数の恒等式、基本統計）をテストする。
3. 開発者が既知の正解と結果を照合する。すべて一致する。
4. 開発者がCalcEngineを宿題チェッカーに統合する。学生が数式を入力すると、アプリがCalcEngineに送信し、結果が期待される答えと比較される。
5. 無料ティアで初期の教室パイロット（学生500名、月間約8,000コール）を処理する。パイロットが全学区に拡大したとき、開発者はStarterプランにアップグレードする。

**成果**：EdTech企業は数学パーサーを構築することなく、宿題チェック機能をリリースします。
**全体ビジョンとの差異**：学生向けのステップバイステップの解法なし、複素数サポートなし、高度なコースの微積分関数なし。

### MVPの制約と前提

- **前提**：開発者は数学演算にローカルライブラリよりもホスト型APIを好む。**外れた場合のリスク**：正確な計算にもかかわらず採用が低迷する。**軽減策**：無料ティアで低コミットメントの検証を可能にする。APIモデルが失敗した場合はオープンソースライブラリモデルに転換する。
- **前提**：式の評価（文字列入力→結果出力）が最も価値の高い機能である。**外れた場合のリスク**：開発者が実際には式の解析よりも個別の関数エンドポイントを好む。**軽減策**：MVPには式の評価と個別の関数エンドポイントの両方が含まれる。どちらが好まれるかは使用状況データで判明する。
- **前提**：IEEE 754倍精度でMVPユーザーには十分である。**外れた場合のリスク**：金融や科学の初期採用者がすぐにより高い精度を要求する。**軽減策**：任意精度はフェーズ2の優先事項であり、需要シグナルが強ければ前倒しできる。
- **前提**：月10,000コールの無料枠は製品評価には十分だが、有料転換を促すのに低すぎない。**外れた場合のリスク**：無料ティアが寛大すぎる（転換なし）か、制限的すぎる（評価前にユーザーが離れる）。**軽減策**：2か月時点での転換データに基づいて上限を調整する。
- **受け入れ済みの制限**：MVPではクライアントSDKなし。開発者は生のHTTPコールを行う必要がある。これは摩擦を増やすが、製品市場適合が検証される前に複数の言語でSDKを構築・保守するのは費用がかかる。
- **受け入れ済みの制限**：シングルリージョン展開（US-East-1）。ヨーロッパやアジアのユーザーのレイテンシは高くなる。計算ペイロードが小さい（帯域幅への影響が低い）ため、MVPとして許容される。

### MVPの完了定義

- [ ] 「必須」とされる13の機能がすべて実装、テスト、デプロイ済み
- [ ] APIがすべてのMVP機能をカバーする500件以上の数学的テストケースの検証スイートに正しく応答する
- [ ] エッジケースが適切に処理される：ゼロ除算、オーバーフロー、アンダーフロー、不正な式、定義域エラー（例：log(-1)）
- [ ] 一般公開前の2週間のバーンイン期間でAPI稼働率99.9%を実証
- [ ] バーンイン期間中の計測でp50レスポンスタイム50ms未満、p99 200ms未満
- [ ] エンドポイントリファレンス、コード例（Python、JavaScript、cURL）、インタラクティブサンドボックスを備えたドキュメントポータルが公開済み
- [ ] 課金統合が機能している：無料ティアが適用され、有料ティアが購入可能で、使用量が正確にトラッキングされている
- [ ] OpenAPI 3.x仕様が公開されダウンロード可能
- [ ] セキュリティレビュー完了：APIキー認証、レート制限、入力バリデーション、インジェクションの脆弱性なし
- [ ] 負荷テスト完了：APIが1,000件の同時リクエストを劣化なく処理する
- [ ] ステークホルダーデモ完了および承認済み

## リスクと依存関係

### 主なリスク

| リスク                                                                             | 可能性   | 影響     | 軽減策                                                                                                                                                                          |
| -------------------------------------------------------------------------------- | ----------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 開発者採用が低迷：市場が数学のホスト型APIよりローカルライブラリを好む            | 中          | 高       | 無料ティアで参入障壁を下げる。クロスプラットフォームの一貫性と式の評価を差別化要素として強調する（ローカルライブラリには欠けている点）。サインアップからアクティブ利用への転換率を監視する。 |
| 精度バグが信頼を損なう：誤った計算結果が公に報告される                           | 低          | 重大     | 包括的なテストスイート（関数ごとに500件以上）、参照実装との比較（Wolfram Alpha、Python mpmath）、デプロイごとの自動化された回帰テスト。                                            |
| 式パーサーのエッジケース：予期しない入力がクラッシュや誤結果を引き起こす          | 中          | 高       | ランダムな式を使ったファズテスト、明示的な文法定義、インジェクションを防ぐためのパーサーのサンドボックス化。                                                                      |
| 無料ティアの悪用：ボットやスクレイパーが転換なしにリソースを消費する              | 中          | 中       | APIキーごとのレート制限、サインアップ時のCAPTCHA、使用パターンの異常検知。必要に応じて無料ティアの上限を調整する。                                                                |
| Stripe課金統合によるMVPローンチの遅延                                            | 低          | 中       | 2か月目に課金統合を開始する。必要に応じて手動請求を一時的なフォールバックとして使用する。                                                                                        |
| シングルリージョン障害によるサービス全体のダウン                                  | 低          | 高       | US-East-1内の2つのアベイラビリティゾーンにデプロイ。マルチリージョンはフェーズ2だが、AZ冗長性でベースラインの回復力を確保する。                                                  |
| 競合他社が開発中に同様のAPIをローンチする                                         | 低          | 中       | 市場投入速度が主な防御策。6か月のMVPタイムライン。ドキュメントの品質、レスポンスタイム、エラーメッセージを競争優位として開発者体験に注力する。                                    |

### 外部依存関係

- **Stripe** - 決済処理とサブスクリプション管理 - 利用可能、ドキュメントも充実したAPI
- **Auth0またはAWS Cognito** - 開発者認証 - 利用可能、1か月目に評価
- **ドメインレジストラ** - calcengine.ioドメイン（または類似） - ドキュメントサイト公開前に確保する必要あり
- **クラウドプロバイダー（AWS）** - コンピューティング、API Gateway、データベース - 利用可能、承認不要
- **SSL証明書プロバイダー** - APIとドキュメントサイト向けのTLS - AWS Certificate Manager経由で利用可能
